INTRODUCTION                                                     *pairagen*

Pairagen presents one hypothesis, finding, or editable patch at a time. It is
designed for explicit user-controlled pair-programming steps rather than an
open-ended chat session.

In the default auto mode, one prompt starts a persistent goal loop. The agent
inspects the project and proposes one editable hunk. Accepting it immediately
continues to the next hunk; rejecting it immediately requests a rework. The
loop ends only with a completed-goal summary. Navigation between files inside
the workspace is automatic, while every code change remains gated by explicit
Accept or Reject. Use `keymaps.why` (`<leader>pw` by default) on a draft to ask
why; the answer keeps that exact hunk pending and offers `Back to draft`.

Long goals and patch explanations are compact by default. Focus the Pair card
and press `z` to expand or collapse the full text. Configure this with
`keymaps.details`.

The session token total is shown against `backend.token_budget` (50000 by
default). Pair asks for confirmation before another agent turn after that
budget is reached. Set it to 0 to disable the guard.

Pairagen has been tested primarily with the Codex CLI app-server backend.

===============================================================================
SETUP                                                       *pairagen-setup*

Minimum setup: >lua

  require("pair").setup({
    backend = { agent = "codex" },
  })
<
The matching `paird` release binary is downloaded and checksum-verified on the
first request. Set `backend.command` to use a manually installed binary.

===============================================================================
COMMANDS                                                 *pairagen-commands*

                                                       *:Pair*
:Pair                  Open a new Pair prompt.
                                                       *:PairReply*
:PairReply             Send a message in the active session.
                                                       *:PairFix*
:PairFix               Request one local editable patch.
                                                       *:PairFollow*
:PairFollow            Follow the current observation.
                                                       *:PairWhy*
:PairWhy               Ask for the reasoning behind the current observation.
                                                       *:PairOther*
:PairOther             Request another lead.
                                                       *:PairAssess*
:PairAssess            Assess the whole goal and locate one unresolved step.
                                                       *:PairNext*
:PairNext              Compatibility alias for |:PairAssess|.
                                                       *:PairStop*
:PairStop              Finish the active session.
                                                       *:PairHide*
:PairHide              Hide the current Pair UI.
                                                       *:PairResume*
:PairResume            Show and focus the current Pair UI.
                                                       *:PairReset*
:PairReset             Stop the backend and clear local session state.
                                                       *:PairAgent*
:PairAgent [name]      Show or select the active agent.
                                                       *:PairModel*
:PairModel [model]     Show or select the active model. Use `default` to clear
                       a persisted model choice.
                                                       *:PairLog*
:PairLog               Print the current local trace path.
                                                       *:PairLogClear*
:PairLogClear          Delete Pairagen session traces.

===============================================================================
PRIVACY                                                   *pairagen-privacy*

Session traces are local, content-redacted by default, and subject to retention.
Enabling `logging.include_content` may store proprietary source code, prompts,
diffs, and model responses. Pairagen sends selected source context to the backend
configured by the user.

===============================================================================
HEALTH                                                     *pairagen-health*

Run: >vim

  :checkhealth pair
<
The report covers Neovim compatibility, release target, managed `paird`,
download verification tools, active agent/model, logging privacy, and LSP
clients.
